Modelo OSI

1. Capa Física:

  • Mecanismos: Controlan el acceso físico a los medios (cables, switches, etc.).
  • Protocolos de seguridad: Ninguno directamente en esta capa, pero se asegura con medidas como seguridad física en centros de datos.
  • Servicios: Control de acceso a equipos y monitoreo físico.

    • 2. Capa de Enlace de Datos:

  • Mecanismos: MAC (Media Access Control).
  • Protocolos de seguridad: 802.1X (autenticación de dispositivos), VLANs.
  • Servicios: Evita ataques de red como ARP spoofing
  • Ejemplo en AWS: Seguridad a nivel de subredes en VPC (Virtual Private Cloud).

    • 3. Capa de Red:

  • Mecanismos: Gestión de direccionamiento y enrutamiento.
  • Protocolos de seguridad: IPSec, GRE, NAT.
  • Servicios: Protege la integridad de los datos durante el enrutamiento.
  • Ejemplo en AWS: Gateways de VPN, control de rutas en VPC.

    • 4. Capa de Transporte:

  • Mecanismos: Gestión de sesiones, confiabilidad de la transmisión.
  • Protocolos de seguridad: TLS/SSL, DTLS.
  • Servicios: Encriptación extremo a extremo, prevención de ataques MITM (Man-In-The-Middle).
  • Ejemplo en AWS: Certificados TLS en AWS Certificate Manager (ACM).

    • 5. Capa de Sesión:

  • Mecanismos: Gestión de sesiones de aplicaciones.
  • Protocolos de seguridad: SMB Signing, TLS Handshake.
  • Servicios: Protege sesiones activas contra intercepción o secuestro.
  • Ejemplo en AWS: Manejo de sesiones para servicios como AWS SSO (Single Sign-On).

    • 6. Capa de Presentación:

  • Mecanismos: Transformación y encriptación de datos.
  • Protocolos de seguridad: AES, RSA, DES.
  • Servicios: Proporciona confidencialidad y formatos de datos seguros.
  • Ejemplo en AWS: Servicios de cifrado en tránsito y en reposo, como AWS KMS (Key Management Service)./li>

    • 7. Capa de Aplicación:

  • Mecanismos: Interacción usuario-aplicación.
  • Protocolos de seguridad: HTTPS, OAuth, SAML.
  • Servicios: Autenticación, autorización, auditorías.
  • Ejemplo en AWS: Autenticación en AWS Cognito, monitoreo con AWS CloudTrail.

    • Comparación con el Modelo TCP/IP

    El modelo TCP/IP tiene menos capas (4), pero mapea directamente a las funciones del OSI

    Modelo OSI Modelo TCP/IP Protocolos de Seguridad Ejemplo en AWS
    Aplicación (7, 6, 5) Aplicación HTTPS, OAuth, SAML, DNSSEC AWS IAM, Cognito, ACM
    Transporte (4) Transporte TLS, DTLS TLS en servicios ELB (Elastic Load Balancing)
    Red (3) Internet IPSec, GRE AWS VPN, Security Groups en VPC
    Enlace de datos y Física (2, 1) Acceso a Red 802.1X, VLANs, PPP Seguridad física en centros de datos, Direct Connect

    Soluciones en Proveedores de Nube (AWS)

    1. Protocolos de Seguridad en AWS

  • TLS/SSL: Para proteger aplicaciones web mediante CloudFront y ACM.
  • IPSec: Usado en túneles VPN de Site-to-Site.
  • OAuth y SAML: En integración con servicios de autenticación federada.
  • DNSSEC: Implementado en Route 53 para proteger la resolución DNS.
